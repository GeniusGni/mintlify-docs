## Project list

The project list provides comprehensive tools for organizing and managing your projects with the following features:

### Filtering and sorting

You can filter and sort projects by:

- Project type
- Owner
- Creation time
- Last modified time

### Project icons

Projects display different icons based on their source:

- **GitLab projects**: GitLab icon
- **GitHub projects**: GitHub icon
- **Local projects**: Folder icon

### Automatic grouping

When you clone multiple projects from the same GitLab branch, they are automatically grouped under a single top-level project entry.

## Project operations

Access additional operations through the "..." menu button on each project.

### Top-level repository projects

Top-level repository projects (folders merged from the same code branch) cannot be deleted.

### Second-level and local projects

You can perform the following operations:

- Rename
- Create new file
- Create new folder
- Upload file
- Upload folder
- Delete project

### Folder operations

Within a project, folders support:

- Rename
- Create new file
- Create new folder
- Upload file
- Upload folder
- Copy relative path
- Download folder
- Copy
- Move
- Delete folder

### File operations

Within a project, files support:

- Rename
- Copy relative path
- Download file
- Copy
- Move
- Delete file

### Copy relative path

The "Copy relative path" feature copies the current file's path relative to a base directory (typically the project root, workspace root, or current folder).

### Permission controls

All delete operations require permission verification. Only users with agent permissions can delete projects, folders, or files.

## Message Notifications

- Unread messages and mention messages will be displayed in chat, notifications, and projects.
- Mention messages will be displayed with higher priority than unread messages.
